<b>Definition.</b><br/>A coding list as specified by a standardisation authority.<br/><br/><b>Remark.</b><br/>The CodeListCode attribute is of the type CodeType, which on its turn also has a CodeListCode, which makes this an endless reciprocal relation. <br/>This should be limited to one.<br/><br/>This means that there is only one authority which keeps a CodeList of CodeList's. For the Netherlands this is AgroConnect.<br/><br/>This does not mean that only AgroConnect makes CodeList's. But AgroConnect can give a code for an ISO codelist.<br/><br/>DO WE NEED AN EXAMPLE ?<br/>
