Changes made on 2012-08-29

SendingParty added to the model. This is done because the IssuingParty is not always the same as the Sendingparty.

Address is moved as AddressType to the datatype package. there is one subclass, DeliveryAddressType which has a GlobalLocationNumber as addissional attribute. Address has the characteristics of a datatype as explained in the modeling guidelines.
Phone, e-mail and webpages are separate datatypes. PhoneNumberType is added as type, with three fields. There is no need to define an e-mail type or a web-site type, as those can be represented by the datatype "anyURI".
As a consequence of the difference made in the modeling guideline between datatypes and normal classes, post address, delivery address, visting address, phone, e-mail en web-page become attributes of the object they belong to. They are assigned as attributes of Party, Person or Organization following the logic of the attribute (Private... as attribute of Person, etc.)
Fax is removed from the model (Who uses it nowedays?)
