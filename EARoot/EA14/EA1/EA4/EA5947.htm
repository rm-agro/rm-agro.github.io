<b>Description.</b><br/>A <b>Task</b> is the execution of one or more <b>Operation'</b>s.<br/><br/><b>Remark.</b><br/>An <b>Operation</b> can be executed by more <b>Task</b>s.(See example below). Therefore, there is a many to many relation between Task and Operation. <br/><br/><b>Example.</b><br/>There is one <b>Task</b> "Harrowing and Drilling", which performs two <b>Operation</b>'s; one is "Harrowing" and the other is "Drilling".<br/>On the other hand, there can be two <b>Task</b>'s that realize one <b>Operation</b>. For example two combines harvesting on one <b>ActivityField</b>.<br/>

Notes: <b>Definition.</b><br/>Date and time at which the Task is started for the first time.<br/><b>Remark.</b><br/>The time must include the time zone information, when no UTC time is used. In the netherlands that is +02:00 during summertime and +01:00 during wintertime.<br/><br/><b>Remark.</b><br/>For a planned Task it is the date at which the Task is intended to start.<br/>When a Task is started again after an interruption, the first starting time should be held.<br/><br/><b>Remark.</b><br/>EarliestBeginTime and LatestEndTime indicate the total elapsed time. Within this period there are one or more time periods during which the Task will/can be executed.<br/>
